PDF to Markdown

Upload a PDF and get a clean, editable Markdown (.md) file. Headings, bullet lists and tables are preserved where possible.

How it works

  1. Text and layout are extracted from the PDF.
  2. Headings are detected and converted to `#` / `##` style Markdown.
  3. Lists and simple tables are converted to `-` lists and pipe tables.

Key Features

  • Preserves headings, lists and basic tables
  • Produces GitHub-friendly Markdown
  • Downloadable `.md` output

Use-cases

Documentation authors, students, and developers converting PDFs for GitHub, Notion, or static sites.

Example output

# Title of Document This is a paragraph extracted from the PDF. ## Section heading - Bullet item 1 - Bullet item 2 | Column A | Column B | |---|---| | Row 1A | Row 1B |

Notes for implementation

  • Server-side: Use `pdfminer.six` (Python) or `pdf.js` (Node) to extract text; then map text blocks to Markdown.
  • For scanned PDFs, add OCR (Tesseract) before parsing.
  • Tables: try `camelot` or `tabula` on server-side to extract accurate tables.
  • Client-side fallback: use `pdf.js` to render pages and extract text if no server component available.
